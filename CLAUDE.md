# Правила проекта CS2 Tournaments

Общие правила разработки. Соблюдай их при любых изменениях в проекте.

## Читаемость и размер файлов
- **В файлах не должно быть много кода — держи их короткими и читаемыми.**
  Если кода становится много, **выноси части в отдельные компоненты** (или в
  composables / утилиты для логики).
- Ориентир: если Vue-компонент разросся (условно больше ~150 строк) или совмещает
  несколько независимых блоков UI — раздели его на под-компоненты.
- Один файл — одна ответственность. Крупные `<template>` разбивай на компоненты,
  повторяющиеся куски выноси в переиспользуемые компоненты.
- Тяжёлую бизнес-логику из компонентов выноси в `app/composables/` или `app/utils/`.

## Структура
- Клиент: `app/` (components, pages, layouts, middleware, composables, utils).
- Сервер: `server/` (api, db, utils). Логику работы с данными держи в `server/utils/`,
  а обработчики `server/api/**` — тонкими.
- Компоненты — в `app/components/`, переиспользуемые и с понятными именами.

## Стиль кода
- Vue 3 только через `<script setup lang="ts">` и Composition API.
- TypeScript везде, где возможно; избегай `any` без необходимости.
- Стили — Tailwind-классы; общие токены темы берём из `app/assets/css/main.css`.
- Комментарии и текст интерфейса — на русском.

## Данные и API
- Компоненты не ходят в БД напрямую — только через API (`/api/**`).
- Вся работа с хранилищем — через `server/utils/repo.ts` (MySQL / демо-хранилище).
- Изменяющие эндпоинты (POST/PATCH/DELETE) обязательно защищаем `requireAdmin`.

## Прочее
- Не коммитим секреты; настройки — только через `.env`.
- Перед завершением задачи проверяй, что dev-сервер запускается без ошибок.

## Деплой
- Пуш в `master` сам запускает автодеплой на прод (GitHub Actions → VPS).
- После пуша не нужно отдельным сообщением отчитываться об успешном деплое —
  можно тихо проверить статус в фоне. Писать пользователю, только если деплой
  реально упал или сайт недоступен.
